
# 1. [hurl 7.0.0](https://github.com/Orange-OpenSource/hurl/releases/tag/7.0.0)  
Hurl, run and test HTTP requests with plain text.

Hurl 7.0.0 版本于2025年7月28日发布，感谢多位贡献者的参与。该版本包含多项重要更新：

主要变更：
- 新增 replaceRegex 过滤器并修复 replace 过滤器不再支持正则表达式

功能增强：
- 新增 HTTP 重定向查询功能
- 新增 urlQueryParam 过滤器
- 在长错误格式中显示 curl 命令
- 支持配置每个请求的最大超时时间
- 新增日期比较谓词
- 添加 pinnedpubkey 命令行选项
- 新增 base64 URL 安全编解码过滤器
- 支持解析 curl 的 --cookie 标志
- 为帮助信息添加彩色显示
- 新增 toHex 过滤器
- 添加 first 和 last 过滤器
- 移除 hex crate 依赖
- 改进测试进度输出显示
- 支持 nth 过滤器的负值参数
- 新增 UUID 验证谓词
- 改进捕获错误的显示信息
- 新增 NTLM 和协商认证选项

问题修复：
- 修复 POST 重定向时的 curl 命令错误
- 修复 curl 命令中无效头参数的问题
- 修复秘密变量覆盖问题
- 修复 zsh 自动补全功能
- 修复 HTML 报告中的标签问题
- 修复重定向时的请求体问题
- 支持 JSONPath 中的负索引

安全修复：
- 修复 HTML 报告中通过正则表达式实现的 JavaScript 注入漏洞

# 2. [dify 1.7.1](https://github.com/langgenius/dify/releases/tag/1.7.1)  
Production-ready platform for agentic workflow development.

Dify v1.7.1 版本发布，带来多项功能优化和问题修复。主要更新内容：

1. 新增功能：
- 支持为选择输入字段设置默认值，提升表单用户体验
- 在列表操作的条件过滤中支持选择变量
- OpenAPI Schema 新增对 allOf 属性的支持
- K8s 部署中为 api 组件添加纯迁移选项

2. 问题修复：
- 修复 Langfuse 集成路径错误
- 解决 RabbitMQ 批量添加文档片段时的处理问题
- 修正跨页文档选择时的元数据批量编辑问题
- 修复 Windows 系统下 PEM 密钥路径错误

3. 改进优化：
- 优化菜单中 ToolTip 组件的交互效果
- 增强 PostgreSQL 健康检查命令
- 重构国际化时间格式化功能
- 改进工具列表页面设计

4. 升级指南：
- Docker Compose 部署：需备份数据后更新代码并重启服务
- 源码部署：需停止服务后更新代码，迁移数据库并重启

本次更新包含大量细节优化，涉及前端交互、API设计、国际化支持等方面，共合并了50多项PR，并有8位新贡献者加入。完整变更日志可查看项目GitHub页面。

Higress Console项目：
1. 发布概述
- 共12项更新，包括6项新功能、5项错误修复和1项重构优化

2. 主要更新
- 支持单条路由配置多个子路由
- 新增多个插件的JSON Schema定义
- 添加DB MCP Server工具管理功能
- 修复LLM提供商管理界面问题、WASM镜像URL空格问题等

3. 优化改进
- 统一项目文件换行符为LF格式
- 移除数据库配置中主机和端口字段的禁用状态
- 更新API网关URL显示逻辑

两个项目均对贡献者表示感谢，持续提升系统功能和用户体验。
