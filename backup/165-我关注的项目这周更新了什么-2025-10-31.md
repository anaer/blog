# 4. [dify 1.9.2](https://github.com/langgenius/dify/releases/tag/1.9.2)  
Production-ready platform for agentic workflow development.

### 主要变更说明

- **稳定性与性能提升**：本次更新着重于提高稳定性、异步性能和开发者体验。代码进行了优化，内部结构更加整洁，工作流控制更加流畅，整个堆栈的可观测性也得到了提升。

- **Weaviate 集成更新**：现代化的 Dify 集成已更新到 Weaviate 4.0 版本，并要求 Weaviate 服务器版本至少为 1.24.0。此更新包括：
  - 升级 Weaviate Python 客户端至 v4。
  - 提高代码的可读性和性能，使用 gRPC 而非 HTTP。
  - 数据迁移可能需要重新索引或使用备份/恢复流程。

- **工作流与代理**：
  - 新增工作流图执行的暂停与恢复功能。
  - LLM 节点流式处理期间支持结构化输出。
  - 代理变量支持拖放操作，与工作流启动节点类似。
  - 可通过日志重新执行或过滤工作流运行状态。

- **集成与SDK**：
  - 支持 OpenTelemetry + HTTPX 追踪，提高可观测性。
  - CORS 配置支持自定义头。

- **Web与UI**：
  - 加快加载时间，通过拆分和懒加载常量文件。
  - 改进数据源，集成市场插件和过滤功能。
  - 添加税率提示到定价页脚。

- **核心改进**：
  - 启用 Pyright 并修复多个模块的类型问题。
  - 优化 HTTP 超时配置和输入验证。
  - 通过缓存检查和显式键清理提高 Redis 队列效率。

- **修复**：
  - 修复重复片段和数据集分页重复问题。
  - 修复工作流令牌使用、LLM 使用跟踪和分离用户会话问题。
  - 解决 Weaviate 向量距离计算的逻辑错误。
  - 修复多认证凭证冲突。
  - 修复聊天闪烁、无限重载、登录重定向和加载指示器可见性问题。

- **清理与开发体验**：
  - 移除未使用依赖、冗余数据库提交和死模板。
  - 添加 Knip 配置以检测死代码。
  - 改进内部可观测性，通过 HTTPX 追踪和异步遥测。

### 升级指南

- **Docker Compose 部署**：
  - 备份自定义的 docker-compose.yaml 文件。
  - 获取最新代码。
  - 停止服务。
  - 备份数据。
  - 升级服务。

- **源代码部署**：
  - 停止 API 服务器、Worker 和 Web 前端服务器。
  - 获取最新代码。
  - 更新 Python 依赖。
  - 运行迁移脚本。
  - 重新运行 API 服务器、Worker 和 Web 前端服务器。

### 代码变更

- 修复了多个 bug，提高了性能。
- 添加和改进了多项功能。
- 优化了内部结构和代码质量。

# 5. [winmerge v2.16.52](https://github.com/WinMerge/winmerge/releases/tag/v2.16.52)  
WinMerge is an Open Source differencing and merging tool for Windows. WinMerge can compare both folders and files, presenting differences in a visual text format that is easy to understand and handle.

### 主要变更

- **通用**
  - 修复了临时文件夹未在下次应用退出时正确删除的问题。
  - 添加了Tab导航的替代快捷键。
  - ARM64构建中禁用了darkmodelib，因存在未解决的崩溃问题。

- **二进制比较**
  - 修复了在十六进制视图中Ctrl-W关闭窗口的问题。

- **文件夹比较**
  - 修复了添加排除项时的错误。
  - 添加了toDateStr过滤函数。
  - 添加了指定额外比较条件的选项。
  - 添加了prop、leftprop、middleprop和rightprop函数支持。
  - 添加了“Existence”文件夹比较方法。

- **翻译**
  - 更新了多种语言的翻译，包括巴西语、中文简体、日语、韩语、立陶宛语、葡萄牙语、波兰语、土耳其语和俄语。

### 已知问题
- 在选项窗口中按下OK可能会损坏正在比较的INI文件。
- 图像比较结果的可靠性有待提高。
- 比较大文件时可能会崩溃。
- 比较显著不同的目录时速度非常慢。
- 粘贴文本后垂直滚动条可能会损坏。
